

# 📡 ESP-NOW to OSC Smart Router

A lightweight, zero-hardware bridge that turns your computer's native Wi-Fi card into a direct ESP-NOW receiver.

Instead of requiring a dedicated "Receiver ESP32" plugged into your computer via USB, this script sniffs raw 2.4GHz radio waves, identifies packets coming from Espressif chips, perfectly slices out **OSC (Open Sound Control)** payloads, and forwards them instantly over UDP to your visual or audio software (Resolume, TouchDesigner, Max/MSP, etc.).

## ✨ Key Features

* **Zero-Hardware Receiver:** Intercepts ESP-NOW traffic directly out of the air using standard Wi-Fi hardware in Monitor Mode.
* **Smart OSC Routing:** Automatically detects valid OSC Messages (`/`) and OSC Bundles (`#bundle`), strips the proprietary Espressif radio headers, and fires the clean bytes to your target port.
* **Auto-Wrapping (`--wrap-raw`):** If an ESP32 sends plain text instead of OSC, the script can automatically pad and wrap it into a valid OSC message (`/rawdata <string>`) so your receiving software doesn't crash.
* **Live Hardware Discovery:** Maintains an active memory bank of all ESP32s in the room, printing a clean Session Summary of unique MAC addresses when you exit.
* **Cross-Platform Automation:** Runs on macOS (using `libpcap`) and Linux (using `iw`/`ip` to automatically force monitor mode and restore your network on exit).
* **Smart Hostname Resolution:** Targets machines using local hostnames (e.g., `visuals-macpro`). The script automatically handles mDNS resolution, even if you forget to type the `.local` suffix.
* **Offline Capable:** Includes an `update` command to fetch the latest IEEE Espressif MAC registry while on Wi-Fi, saving a local cache so the router can run flawlessly in fully offline environments.

## ⚙️ Prerequisites & Installation

1. Ensure you have **Python 3** installed.
2. Install the required network analysis libraries:
```bash
pip install scapy mac-vendor-lookup

```


3. **Root Privileges:** Because this script interacts directly with raw network hardware, it *must* be run using `sudo`.

---

## 🚀 The Standard Workflow

Because Monitor Mode requires disconnecting from standard Wi-Fi, you must update the script's hardware database *before* you go offline at your venue.

1. **Connect to normal Wi-Fi** (with internet).
2. **Update the database:**
```bash
sudo python3 esp_target_lock.py update

```


3. **Disconnect from Wi-Fi.**
4. **Launch the Router:** (See OS-specific instructions below).

---

## 💻 Usage & Command Line Arguments

The script requires a specific positional command structure:
`sudo python3 esp_target_lock.py <OS> <TARGET_MAC|ALL> <INTERFACE> <CHANNEL> <DEST_IP_OR_HOST> <DEST_PORT> [LOG.csv] [--wrap-raw]`

### Argument Breakdown

| Argument | Description | Examples |
| --- | --- | --- |
| **OS** | Your current operating system. | `mac`, `osx`, `linux` |
| **TARGET** | The MAC address of the ESP32 to listen to, or `ALL` to track every Espressif device in the room. | `ALL`, `24:0a:c4:11:22:33` |
| **INTERFACE** | Your Wi-Fi network interface name. | `en0` (Mac), `wlan1` (Linux) |
| **CHANNEL** | The 2.4GHz Wi-Fi channel your ESP32s are broadcasting on. (Use `0` on Linux for Scan Mode). | `1`, `6`, `11`, `0` |
| **DESTINATION** | Where to send the OSC UDP packets. Accepts IP addresses or hostnames. | `127.0.0.1`, `visuals-macpro` |
| **PORT** | The UDP port your visual/audio software is listening on. | `8000`, `9000` |
| **[LOG_FILE]** | *(Optional)* Saves a complete, timestamped CSV history of all intercepted packets. | `session_log.csv` |
| **[--wrap-raw]** | *(Optional)* Flag to wrap non-OSC text into an OSC `/rawdata` packet. Can be placed anywhere. | `--wrap-raw` |

## 🍏 macOS Specifics

Because Apple heavily restricts background hardware control, the script uses `libpcap` to read the raw radio waves, but you must manually lock the frequency.

1. Disconnect from your Wi-Fi network.
2. Hold **Option (⌥)** and click the Wi-Fi icon in your top menu bar. Click **Open Wireless Diagnostics...**
3. In the top menu bar, click **Window > Sniffer**.
4. Set the channel to match your ESP32 (e.g., `1`) and click **Start**.
5. Run the script in your terminal:
```bash
sudo python3 esp_target_lock.py mac ALL en0 1 visuals-macpro 8000

```



---

## 🐧 Linux / Raspberry Pi Specifics

Linux allows Python to fully control the hardware. The script will automatically tear down your managed Wi-Fi, force the adapter into Monitor Mode, lock the channel, and restore everything to normal the moment you press `Ctrl+C`.

**Standard Single-Channel Lock:**

```bash
sudo python3 esp_target_lock.py linux ALL wlan1 1 visuals-macpro 8000 session_data.csv --wrap-raw

```

**Scan Mode (Channel 0):**
If you don't know what channel your devices are on, pass `0` as the channel argument. The script will launch a background thread that continuously sweeps channels 1, 6, and 11 to discover lost devices.

```bash
sudo python3 esp_target_lock.py linux ALL wlan1 0 visuals-macpro 8000

```

*Note: Scan mode will inherently drop packets as the radio physically hops between frequencies. Once devices are discovered, restart the script locked onto their specific channel for zero-packet-loss routing.*

## 📦 Payload Processing & Gatekeeping

This router acts as a strict gatekeeper to protect your receiving software from crashing on garbage network noise.

* **Valid OSC:** If a packet contains a standard OSC address (`/`) or bundle (`#bundle`) paired with a valid null-terminated type tag, the script strips the radio header and forwards the raw bytes.
* **Raw Data:** If an ESP32 sends normal text (e.g., `"temp: 24.5"`):
* If `--wrap-raw` is **OFF**: The script prints the data to the terminal and CSV log, but safely drops it from the UDP queue.
* If `--wrap-raw` is **ON**: The script mathematically pads the string with null bytes, prepends the `/rawdata` address, and routes it as a valid OSC string message.
